## Overview

DefinitionTooltip clarifies the meaning of technical terms or jargon by displaying a definition on hover or focus. It underlines the term with a dotted line to indicate additional information is available, helping users understand unfamiliar terminology without leaving their current context.

## When to Use

- To define technical terms, acronyms, or jargon
- When users may not be familiar with specific terminology
- For providing contextual explanations without cluttering the interface
- In documentation, help text, or educational content
- When a brief definition is sufficient (not long explanations)

## When Not to Use

- For interactive help or detailed information (use Toggletip instead)
- When the definition is very long (use a link to dedicated page)
- For actionable content or forms (use Toggletip)
- When the term is commonly understood by your audience
- For critical information that all users must see

---

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `definition` | `node` | required | Definition text to display in tooltip |
| `children` | `node` | required | Term to be defined |
| `align` | `'start' \| 'center' \| 'end'` | `'start'` | Tooltip text alignment |
| `className` | `string` | - | Additional CSS classes |
| `openOnHover` | `boolean` | `true` | Open tooltip on hover |
| `triggerClassName` | `string` | - | CSS classes for the trigger element |

### In a Glossary

```jsx
import { DefinitionTooltip } from '@carbon/react';

function GlossaryTerm({ term, definition }) {
  return (
    <li>
      <DefinitionTooltip definition={definition}>
        {term}
      </DefinitionTooltip>
    </li>
  );
}

function Glossary() {
  const terms = [
    { term: 'SaaS', definition: 'Software as a Service: cloud-based software delivery model' },
    { term: 'CI/CD', definition: 'Continuous Integration/Continuous Deployment: automated development practices' },
    { term: 'SDK', definition: 'Software Development Kit: tools for building applications' }
  ];

  return (
    <ul>
      {terms.map(({ term, definition }) => (
        <GlossaryTerm key={term} term={term} definition={definition} />
      ))}
    </ul>
  );
}
```

## Accessibility

### Keyboard Support

- **Tab**: Move focus to the definition term
- **Enter/Space**: Open tooltip (when `openOnHover` is false)
- **Esc**: Close tooltip
- **Hover**: Automatically shows tooltip

### Screen Reader Support

- The term is announced with its definition via `aria-describedby`
- Tooltip content is accessible to screen readers
- The underline styling provides a visual cue that's not solely dependent on color

### Best Practices

- Keep definitions concise (1-2 sentences maximum)
- Use clear, simple language in definitions
- Only use for terms that truly need clarification
- Don't overuse on a single page (max 10-15 terms)
- Ensure sufficient color contrast for the underlined text
- Test with keyboard-only navigation
- Avoid defining terms that are already well-known to your audience
