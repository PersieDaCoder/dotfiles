## Overview

The AILabel component is used to indicate AI-generated or AI-assisted content and features. It provides a consistent way to communicate when artificial intelligence is being used within an interface, helping maintain transparency with users.

## When to use

- To label AI-generated content or responses
- To indicate AI-assisted features or functionality
- When displaying content that has been created, modified, or suggested by AI
- To maintain transparency about AI usage in your application
- For compliance with AI disclosure requirements

## When not to use

- For non-AI related labels (use Tag or Label instead)
- When AI involvement is not relevant to the user experience
- For decorative purposes only

## Usage

### Basic Example

```jsx
import { AILabel } from '@carbon/react';

function MyComponent() {
  return (
    <AILabel>AI</AILabel>
  );
}
```

### With Custom Text

```jsx
<AILabel>AI Generated</AILabel>
```

### Inline Usage

```jsx
<div>
  <h3>
    Product Description <AILabel size="sm">AI</AILabel>
  </h3>
  <p>This product description was generated using AI...</p>
</div>
```

### Different Sizes

```jsx
{/* Extra Small */}
<AILabel size="xs">AI</AILabel>

{/* Small */}
<AILabel size="sm">AI</AILabel>

{/* Medium (default) */}
<AILabel size="md">AI</AILabel>

{/* Large */}
<AILabel size="lg">AI</AILabel>
```

### With TextInput

```jsx
import { TextInput, AILabel } from '@carbon/react';

<TextInput
  id="ai-input"
  labelText={
    <>
      Email subject <AILabel size="sm">AI</AILabel>
    </>
  }
  placeholder="Enter subject..."
/>
```

### With Slug (Callout)

```jsx
import { AILabel, AILabelContent, AILabelActions, Button } from '@carbon/react';

<AILabel
  slug={
    <AILabelContent>
      <div>
        <h4>AI Generated Content</h4>
        <p>This content was generated by AI and may need review.</p>
      </div>
      <AILabelActions>
        <Button kind="ghost" size="sm">Review</Button>
        <Button kind="ghost" size="sm">Accept</Button>
      </AILabelActions>
    </AILabelContent>
  }
>
  AI
</AILabel>
```

### Revert Button

```jsx
import { AILabel, IconButton } from '@carbon/react';
import { Undo } from '@carbon/icons-react';

<div className="ai-content-container">
  <AILabel>AI Generated</AILabel>
  <IconButton
    label="Revert AI changes"
    kind="ghost"
    size="sm"
  >
    <Undo />
  </IconButton>
</div>
```

### Kind Variants

```jsx
{/* Default */}
<AILabel kind="default">AI</AILabel>

{/* Inline */}
<AILabel kind="inline">AI</AILabel>
```

## Props

### AILabel

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `align` | `'top' \| 'top-left' \| 'top-right' \| 'bottom' \| 'bottom-left' \| 'bottom-right' \| 'left' \| 'left-bottom' \| 'left-top' \| 'right' \| 'right-bottom' \| 'right-top'` | `'bottom'` | Specify how the popover should align with the trigger element |
| `autoAlign` | `bool` | `false` | Enable auto-alignment of the popover |
| `children` | `node` | - | Content to be displayed in the AILabel |
| `className` | `string` | - | Custom CSS class name |
| `kind` | `'default' \| 'inline'` | `'default'` | Specify the kind of AILabel |
| `onRevertClick` | `func` | - | Callback when revert button is clicked |
| `revertActive` | `bool` | `false` | Specify if the revert button is in active state |
| `revertLabel` | `string` | `'Revert to non-AI content'` | Label for the revert button |
| `showRevertButton` | `bool` | `false` | Show or hide the revert button |
| `size` | `'xs' \| 'sm' \| 'md' \| 'lg'` | `'md'` | Specify the size of the AILabel |
| `slug` | `node` | - | Provide content to be shown in a popover when the AILabel is clicked |
| `slugContent` | `node` | - | **Deprecated.** Use `slug` instead |

### AILabelContent

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `node` | - | Content to display in the AI label popover |
| `className` | `string` | - | Custom CSS class name |

### AILabelActions

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `node` | - | Action buttons to display in the AI label popover |
| `className` | `string` | - | Custom CSS class name |

## Accessibility

- The AILabel is announced to screen readers as "AI" or with the provided text
- The component uses appropriate ARIA attributes for accessibility
- Popover content is accessible via keyboard navigation
- Revert buttons include appropriate labels for screen readers
- The component supports focus management for keyboard users

## Best Practices

- Use AILabel consistently across your application
- Place the label near AI-generated content
- Keep the label text concise (typically just "AI")
- Provide additional context in the popover when needed
- Use the revert functionality when users can undo AI changes
- Consider user's right to know when AI is being used
- Be transparent about AI capabilities and limitations

## Usage Guidelines

### Do

- Use AILabel to clearly indicate AI-generated content
- Provide users with the ability to revert AI changes when applicable
- Place the label prominently but not intrusively
- Use consistent sizing appropriate for the context

### Don't

- Don't overuse AILabel on every element
- Don't hide important information only in the popover
- Don't use for non-AI related features
- Don't make the label the only indication of AI usage in critical situations

## Related Components

- **Tag**: Use for general-purpose labeling
- **Slug**: The underlying popover component
- **Toggletip**: For additional contextual information
- **IconButton**: Often used alongside for revert actions

## Available Variants

- Default
- Inline
- With Popover/Slug
- With Revert Button
- Different sizes (xs, sm, md, lg)
- With form inputs
- With various alignment options
